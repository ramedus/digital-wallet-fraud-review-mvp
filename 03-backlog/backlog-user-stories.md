## Prioritization Approach
- MoSCoW is used to define MVP scope
- RICE is used to compare relative business impact
- Only manual fraud review capabilities are included

---

## User Story List

### US-01: View Suspicious Transaction Queue
As a fraud analyst,  
I want to see all suspicious transactions in a centralized queue,  
so that I can review cases efficiently without missing any transactions.

MoSCoW: Must  
RICE: 80

---

### US-02: Transaction Prioritization by Risk
As a fraud analyst,  
I want suspicious transactions to be automatically prioritized by risk and amount,  
so that high-risk transactions are reviewed first.

MoSCoW: Must  
RICE: 95

---

### US-03: View Transaction Details
As a fraud analyst,  
I want to view all relevant transaction and customer details on a single screen,  
so that I can make faster and more informed decisions.

MoSCoW: Must  
RICE: 85

---

### US-04: Make Fraud Decision
As a fraud analyst,  
I want to approve, block, or escalate a suspicious transaction,  
so that appropriate action can be taken based on my investigation.

MoSCoW: Must  
RICE: 90

---

### US-05: Capture Decision Reason
As a fraud analyst,  
I want to record a decision reason for each reviewed transaction,  
so that decisions are traceable and auditable.

MoSCoW: Must  
RICE: 75

---

### US-06: SLA Tracking for Reviews
As a fraud operations manager,  
I want SLA timers to track review duration,  
so that delayed fraud decisions can be identified and addressed.

MoSCoW: Must  
RICE: 70

---

### US-07: Fraud Review Status Tracking
As a fraud analyst,  
I want to see the current review status of each transaction,  
so that I know which cases are pending, in progress, or completed.

MoSCoW: Must  
RICE: 65

---

### US-08: Basic Fraud Performance Dashboard
As a fraud operations manager,  
I want to see basic metrics such as average decision time and reviewed case count,  
so that I can monitor team performance.

MoSCoW: Should  
RICE: 60

---

### US-09: Escalation Queue
As a fraud analyst,  
I want escalated transactions to appear in a separate queue,  
so that complex cases are handled by senior analysts.

MoSCoW: Should  
RICE: 55

---

### US-10: Fraud Decision History
As a fraud analyst,  
I want to access historical fraud decisions,  
so that I can reference past investigations when needed.

MoSCoW: Should  
RICE: 50
