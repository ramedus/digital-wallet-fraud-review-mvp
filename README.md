# Digital Wallet Fraud Review & Decisioning Module (MVP)

This repository contains a **Business Analyst / Product Owner case study** 
focused on the design of a **Fraud Review & Decisioning MVP** for a digital wallet application.

The goal of this case study is to demonstrate:
- Problem framing and scope definition
- AS-IS / TO-BE process design
- MVP prioritization
- User stories and backlog thinking
- Data modeling and API design (conceptual)

> This is a **conceptual sample project**.  
> No production code, machine learning models, or real financial data are included.

## Case Study Structure
- Problem Statement & Scope
- Process Flows (AS-IS / TO-BE)
- Backlog & User Stories
- Wireframes
- Test Scenarios
- Data Model & API Design
- Metrics & Release Plan
